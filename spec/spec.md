### :
- **Идентификатор**: Уникальный ID тест-кейса (например, BE-1, FE-5, AP-3)
- **Статус**: 
  - `Автоматизирован` - тест выполняется автоматически
  - `Ручной` - требует ручного выполнения
- **Приоритет**:
  - `Высокий` - критически важные функции
  - `Средний` - важные, но не критические функции
  - `Низкий` - второстепенные функции

### Приоритет багов

- **Критический** - Блокирующий дефект, останавливающий работу системы
  - Примеры: падение сервера, потеря данных, невозможность использовать основные функции
  - Время реакции: немедленное исправление

- **Высокий** - Серьёзное нарушение основных функций
  - Примеры: неработающая ключевая функциональность, ошибки в критических сценариях
  - Время реакции: исправление в ближайшем релизе

- **Средний** - Проблема, не нарушающая основную функциональность
  - Примеры: частичная неработоспособность второстепенных функций, UI-дефекты
  - Время реакции: исправление в плановом порядке

- **Низкий** - Незначительные дефекты, не влияющие на функциональность
  - Примеры: опечатки, мелкие косметические недочёты
  - Время реакции: исправление по возможности
